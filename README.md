# Tsovic-Tsov
Tsovic-Tsov — это интерактивный гид по Еревану, превращающий прогулки в культурное приключение. Сканируйте QR-коды на памятниках, чтобы собирать цифровые коллекции и узнавать историю города через увлекательные факты и визуальные материалы. Исследуйте улицы, дополняя физический мир цифровыми артефактами.

В репозитории два приложения:

| Папка | Назначение |
|--------|------------|
| Корень (`App.js`, `src/`) | Мобильное приложение **Expo** |
| `web/` | Сайт **Next.js**: публичный API, данные в JSON, **админка** и выгрузка **QR PNG** |

## Сайт и админка (`web/`)

1. Установите зависимости и задайте секрет админки (или отредактируйте уже созданный локальный файл):

   ```bash
   cd web
   npm install
   cp .env.example .env.local
   ```

   В `.env.local` укажите `ADMIN_SECRET` — длинная случайная строка. Для первого запуска в репозитории может уже лежать `web/.env.local` с dev-секретом (файл в `.gitignore`).

2. Запуск:

   ```bash
   npm run dev
   ```

   Откройте в браузере [http://localhost:3000](http://localhost:3000) — главная с ссылками на **/admin** и **GET /api/figures**.

3. **Админка** (`/admin`): войдите тем же значением, что и `ADMIN_SECRET`. Там можно создавать и править фигуры, задавать **Image URL** для карточки в приложении, скачивать **QR PNG** (в коде QR — **ровно `id` фигуры**, как ожидает сканер в приложении).

4. Данные хранятся в файле **`web/data/figures.json`** (удобно для MVP; позже можно заменить на БД).

## Мобильное приложение (Expo)

```bash
npm install
npx expo start
```

Откройте проект в **Expo Go** на телефоне. Камеру и QR лучше проверять на **реальном устройстве**.

### Загрузка каталога с сайта

1. Узнайте IP компьютера в LAN (например `192.168.10.14`), на котором запущен `web` (`npm run dev` слушает порт **3000**).

2. В **корне** репозитория (рядом с `package.json` приложения Expo) создайте файл `.env` (шаблон — [.env.example](.env.example)):

   ```env
   EXPO_PUBLIC_FIGURES_API_URL=http://ВАШ_IP:3000
   ```

3. Перезапустите Metro (`Ctrl+C`, снова `npx expo start`). На главном экране появится строка «Loaded from server»; при ошибке сети используется встроенный список из `src/data/figures.js`.

После изменений в админке каталог на телефоне **не обновляется сам по себе**: потяните список **вниз** (pull-to-refresh) или снова откройте экран коллекции (например вернитесь с экрана сканера) — приложение заново запросит `/api/figures`.

На Android для `http://` в LAN в `app.json` включён **`usesCleartextTraffic`** (для продакшена лучше HTTPS и выключить cleartext).

### Тестовые QR-коды

В QR должна быть **одна строка** — **id** фигуры из каталога (например `abovyan` или любой id, добавленный в админке). В админке кнопка **QR PNG** скачивает готовый код.
